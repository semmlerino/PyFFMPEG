================================================================================
KEY FINDINGS
================================================================================

Finding #1: qtbot.wait() Timing Anti-Pattern
  - 308 instances of hardcoded timing delays
  - Under CPU contention, delays become insufficient
  - Causes intermittent failures in parallel execution
  - Solution: Use qtbot.waitUntil() or qtbot.waitSignal()

Finding #2: time.sleep() Mostly Acceptable
  - 18 cases in test doubles/mocks (OK)
  - 7 cases in test logic that should be fixed
  - Most have clear documentation

Finding #3: processEvents() Needs Context
  - 25 cases in cleanup/finally blocks (acceptable)
  - 10 cases mid-test without clear purpose
  - Need documentation for clarity

Finding #4: Parallel Execution Impact
  - Tests pass 100% serially
  - May fail intermittently with -n 2
  - Higher failure rate with -n auto
  - Fixes will make parallel execution reliable
