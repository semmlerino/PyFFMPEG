# Drop-in text you can paste into the guide

## New: Test Layout & Conventions

* **Folders:** `tests/unit`, `tests/qt`, `tests/integration`, `tests/e2e`
* **Names:** `test_<module>__<behavior>.py`, test funcs `test_<Class>__<method>__<case>`
* **Rule of thumb:** If it needs `qtbot`, it goes in `tests/qt`.

## New: Flakiness & Order Independence

* Enable `pytest-randomly`, `pytest-timeout`.
* For races, use `qtbot.waitSignal`/`waitUntil` — no bare sleeps.
* `xfail(strict=True)` with an issue # and expiry date.

## New: Hypothesis Invariants

* Use on parsers, path builders, and cache keys.
* Fail a build if any Hypothesis test is flaky across seeds.

## New: CI Environment for Qt

* `QT_QPA_PLATFORM=offscreen`, disable HiDPI, ensure single `QApplication`.
* Never create GUI objects in worker threads (fatal).&#x20;
