The fix contains a CRITICAL BUG that breaks all rez commands.

================================================================================
CRITICAL FINDING
================================================================================

Pattern stripping removes the closing quote from rez commands:

  Input:  rez env nuke -- bash -ilc "nuke /file &"
  Output: rez env nuke -- bash -ilc "nuke /file
          [Opening quote there]          [Missing closing quote!]

Result: bash: unexpected EOF while looking for matching `"'

CODE: terminal_dispatcher.sh line 113
  Removes THREE characters: space, ampersand, and quote!

================================================================================
WHY TEST SUITE PASSES
================================================================================

The test suite ONLY validates pattern stripping syntax.
It does NOT validate that commands execute successfully.

Example:
  Input:  'rez env nuke -- bash -ilc "command &"'
  Output: 'rez env nuke -- bash -ilc "command'
          
  Test Result: PASS (pattern was stripped)
  Execution Result: FAIL (syntax error: unmatched quote)

The test suite EXPECTS and validates the BROKEN behavior!

================================================================================
IMPACT ASSESSMENT
================================================================================

Commands that FAIL:
  - All rez commands (primary launch mechanism)
  - Any command with bash -ilc in it
  - Multi-line quoted commands

Commands that WORK:
  - Direct commands without quotes
  - Simple unquoted commands

User Experience:
  - Rez-based launches fail silently
  - Terminal appears to hang
  - Subsequent commands fail with cascade errors
  - No clear error message to user

================================================================================
ROOT CAUSE
================================================================================

Two components fight over who manages backgrounding:

1. command_launcher.py (lines 451-456):
   Adds ' &"' INSIDE the quoted bash command

2. terminal_dispatcher.sh (line 113):
   Tries to strip it but removes the quote TOO

Result: Design conflict that breaks quote matching
